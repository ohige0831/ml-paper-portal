# Cloudflare + OpenAI nano 契約・設定チェックリスト

## 目的
このチェックリストは、機械学習論文の日本語入口サイトを **Cloudflare + OpenAI nano** 構成で開始するための、契約・初期設定・安全設定の順番を整理したものです。

---

## 0. 事前に確定しておくこと
- [ ] サイトの一言コンセプトを固定する
- [ ] URL方針を固定する
  - [ ] `/papers/<slug>`
  - [ ] `/tags/<slug>`
  - [ ] `/latest`
  - [ ] `/recommended`
- [ ] 対象分野を機械学習系に限定する
- [ ] PDF本体は自前保存しない方針を確定する
- [ ] LLMの役割を「短文要約・導入文生成」に限定する
- [ ] 最初は承認フローあり、その後に自動走行へ移行する方針を確定する

---

## 1. Cloudflare 側の契約・初期設定

### 1-1. アカウント
- [ ] Cloudflare アカウントを作成する
- [ ] 2要素認証を有効化する
- [ ] 支払い方法を登録する
- [ ] Workers Paid を使うか確認する
  - 推奨: 最初から Paid 想定で準備

### 1-2. ドメイン
- [ ] 使用するドメイン名を決める
- [ ] ドメインを取得する
- [ ] Cloudflare にドメインを追加する
- [ ] DNS 管理を Cloudflare 側へ寄せるか決める

### 1-3. Pages / Workers
- [ ] Cloudflare Pages を有効にする
- [ ] Cloudflare Workers を有効にする
- [ ] `workers.dev` でまず試すか、最初から独自ドメインに載せるか決める
- [ ] Cron Trigger を使う前提で Workers プロジェクトを用意する

### 1-4. 保存先の初期方針
- [ ] D1 を使う前提でよいか確認する
  - 用途: 論文メタデータ、要約、タグ、状態管理
- [ ] R2 を使う前提でよいか確認する
  - 用途: 生成済みJSON、静的アセット、補助ファイル
- [ ] KV は最初は最小限に留める
  - 用途: 軽い設定値、キャッシュ

---

## 2. OpenAI 側の契約・初期設定

### 2-1. アカウント
- [ ] OpenAI API を使えるアカウントを準備する
- [ ] 支払い方法を登録する
- [ ] 利用上限を設定する
  - [ ] 月額上限
  - [ ] 低めのしきい値アラート

### 2-2. APIキー
- [ ] APIキーを発行する
- [ ] APIキーをローカルに平文保存しない
- [ ] Cloudflare Secrets に登録する前提にする
- [ ] 開発用と本番用でキーを分けるか検討する

### 2-3. モデル方針
- [ ] 第一候補を `GPT-5.4 nano` にする
- [ ] 品質不足時の第二候補を `GPT-5.4 mini` にする
- [ ] 初期用途を以下に限定する
  - [ ] 一言まとめ
  - [ ] 3行要約
  - [ ] こんな人向け
  - [ ] 難易度
  - [ ] 補助タグ

---

## 3. Cloudflare 側で先に用意するもの

### 3-1. Secrets / Environment
- [ ] `OPENALEX_API_KEY`
- [ ] `OPENAI_API_KEY`
- [ ] 必要なら `SITE_BASE_URL`
- [ ] 必要なら `APP_ENV`

### 3-2. Workers プロジェクト
- [ ] 1つ目: 定期取得・要約生成用 Worker
- [ ] 2つ目は不要か検討
  - 基本は 1 Worker に寄せる
- [ ] Cron Trigger の実行頻度を決める
  - 推奨: 1日1回から開始

### 3-3. D1 スキーマの初期方針
最低限必要なテーブル:
- [ ] `papers`
- [ ] `tags`
- [ ] `paper_tags`
- [ ] `summaries`
- [ ] `jobs` または `fetch_logs`
- [ ] `publish_states`

### 3-4. R2 の初期用途
- [ ] 生成済み JSON
- [ ] 静的生成の中間成果物
- [ ] 後から再生成したい補助データ

---

## 4. OpenAI API 利用の安全設定
- [ ] 最初は日次数十本規模に制限する
- [ ] 1論文あたりの出力項目を固定する
- [ ] 長文要約を禁止する
- [ ] 生成失敗時のリトライ回数を決める
- [ ] 同一論文の再生成条件を決める
  - [ ] 手動差し戻し時のみ
  - [ ] プロンプト改訂時のみ
  - [ ] 自動再生成はしない
